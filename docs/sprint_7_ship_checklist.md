   `residency/manager.py` тАФ it does **not** add the missing module

   Resolution path: either restore the missing modules (out of scope
   for sprint 7 тАФ the task explicitly forbids touching `softbody/`,
   `fluid/`, `sim_field.py`), OR file a documented compat break against
   each of the three games and remove the names from the contract.

2. **Runtime `__version__` is stale.** Update
   `python/slappyengine/__init__.py:4` from `"0.1.0"` to `"0.2.0a0"`
   (or to whatever release number sprint 7 ships under). The test
   added in this sprint will catch any future drift.

## CI workflow audit

Both workflows in `.github/workflows/` enumerated:

| Workflow         | Triggers              | Runs tripwire? | Runs all-demos smoke? | Runs version test? |
| ---------------- | --------------------- | -------------- | --------------------- | ------------------ |
| `ci.yml`         | push + PR to master   | yes (via `pytest tests/ -v` which globs the whole suite) | yes (same glob) | yes (same glob) |
| `publish.yml`    | tags `v*.*.*` only    | no (no test step) | no | no |

**Gap:** `publish.yml` runs wheel-build + sdist + `pypa/gh-action-pypi-publish`
on tag push **without re-running the test suite**. CI already gates `master`
so this is acceptable, but a belt-and-braces add of a `pytest tests/ -v`
step ahead of the wheel build would catch the case where a maintainer
tags an unreviewed commit by mistake. Marked as a nice-to-have, not a
blocker.

No workflow-level changes were required for the new
`SlapPyEngineTests/tests/test_version_consistency.py` тАФ the existing `pytest tests/ -v` in
`ci.yml` picks it up by glob.

## Candidate tag string

Computed by `SlapPyEngineTests/tests/test_version_consistency.py::test_candidate_tag_string`
(prints `CANDIDATE_TAG=<tag>` to stdout). With `pyproject.toml` at
`0.2.0a0` (a pre-release marker), the tag is:

```
v0.2.0a0-candidate
```

The `-candidate` suffix is stripped once the version-drift block above
is reconciled and any pre-release tag (`a`, `b`, `rc`) is bumped to a
final release.
